## Codebase Patterns
- Config directory: ~/.purrfectclaw (used by api-client.ts for config storage)
- CLI routing: main.ts uses switch/case on group then command, with parseArgs() for flags
- Module pattern: ESM with .js extensions in imports, verbatimModuleSyntax enabled
- Typecheck: `npx tsc --noEmit` (tsc not globally installed, use npx)
- GPG signing disabled for commits on this branch (local GPG config issue)
- homedir() from node:os used for home directory paths
- Lock file uses atomic write (write .tmp then rename) to prevent corruption
- Lock file paths: local = ./skills-lock.json, global = ~/.purrfectclaw/skills-lock.json
- Marketplace API: base URL resolved via SKILL_MARKETPLACE_URL env → config 'skill-marketplace-url' → hardcoded default
- Config reading: reuse readConfigFile pattern from api-client.ts (read ~/.purrfectclaw/config.json, return {} on missing/corrupt)
- API client uses native fetch (Node 18+), no auth needed for public marketplace API
- URL params: use encodeURIComponent for path segments, URLSearchParams for query strings
- Canonical skill paths: local = .skills/<slug>/, global = ~/.purrfectclaw/skills/<slug>/
- ZIP extraction: use fflate (unzipSync), always validate against zip-slip before writing
- Symlink with 'junction' type for cross-platform compat; auto-fallback to copy on failure
- Skill command routing: main.ts switch case with dynamic imports, early return pattern (like wallet)
- Stub command handlers accept (args, positional?) — positional extracted via rest.filter(a => !a.startsWith('--'))
- picocolors (imported as `pc`) used for CLI output styling; added as production dependency
- Boolean flags (--remote, --json, --global) parsed as string 'true' by parseArgs, compare with === 'true'
- parseArgs returns Record<string, string> (last value wins) — for multi-value flags use comma-separated: --agent claude-code,cursor
- Install flow: downloadSkill() → installSkill() → upsertLockEntry() — each step can fail independently, wrap in try/catch
- Interactive prompts: use @clack/prompts (text, select, multiselect, isCancel), dynamically imported to keep non-interactive path lightweight
- TTY detection: check process.stdin.isTTY before showing interactive prompts; fall back to usage/error in non-TTY
